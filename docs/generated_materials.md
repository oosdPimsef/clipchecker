# Рабочая папка Clipchecker_materials

`Clipchecker_materials` - это служебная папка одного запуска анализа. Она создается внутри папки с превью-роликом и содержит только сгенерированные программой материалы: кадры, OCR, тексты документов, логи, отчеты и промежуточные файлы.

При каждом новом запуске анализа через `run_analysis.py` папка `Clipchecker_materials` в выбранной папке превью пересоздается. Если такая папка уже существует, программа сначала удаляет старую папку целиком, затем создает новую пустую папку и начинает писать новый `clipchecker.log`.

Это сделано, чтобы результаты повторных запусков не смешивались со старыми артефактами и интерфейс Flask показывал только данные текущего анализа.

Flask после старта фонового процесса коротко ждёт появления нового `clipchecker.log` или `run_status.json`. Это защищает интерфейс от чтения старого лога в момент, когда новый процесс уже начал очищать папку.

Ограничение безопасности: программа удаляет только путь с точным именем `Clipchecker_materials`, который находится непосредственно внутри выбранной папки превью. Исходные папки с превью, документами и эфирной версией не очищаются.
